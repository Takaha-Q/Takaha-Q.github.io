Windowsコマンドラインでファイル名が正しく表示されない場合の対応

Windowsコマンドラインで git status や git diff を実行した場合、日本語などのマルチバイト文字がファイル名に含まれているとそのままでは正しく表示されない。
(本来のファイル名) てすと.html (コマンドラインでの表示) \343\201\246\343\201\231\343\201\250.html
以下のコマンドを実行することで正しく表示されるようになる。
git config --global core.quotepath false
